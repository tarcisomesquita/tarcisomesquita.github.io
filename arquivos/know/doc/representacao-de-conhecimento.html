Penso o conhecimento como sendo o conteúdo da memória. Assim como dividimos nosso conhecimento em coisas, o conteúdo da memória pode ser dividido em arquivos. Cada arquivo representando uma coisa e tendo um endereço na internet. Podemos incluir no texto, junto às palavras, o endereço da coisa que ela refere. Veja um exemplo de texto marcado neste link e sua versão em pdf.

O software Know é para ajudar a identificar sequências de palavras que expressam relações entre entidades presentes numa base de conhecimento. Depois pretendo trabalhar para identificar essas sequências em um texto.

Estrutura

+-----------------------+
| base de conhecimento  |
+-----------+-----------+
            |
     +------+------+
     | sequências  |
     +------+------+
            |
   +--------+--------+
   | base de estados |
   +--------+--------+

Usando as tabelas de uma base de conhecimento, construímos sequências seguindo as relações em uma tabela. Usando as sequências, contruímos a base de estados, que são arquivos json com os possíveis estados de um teclado (teclas ativadas/desativadas) que permite digitar essas sequências.

Base de conhecimento

Assim como usamos uma palavra para representar uma coisa, numa base de conhecimentos usamos um arquivo na internet para representar uma coisa. Atualmente se usa o formato JSON-LD para os arquivos. Podemos pensar o conteúdo do arquivo como sendo uma tabela.

Arquivos que ajudam a entender uma base de conhecimento:

  • top-level0.ttl
  • top-level.jsonld
  • top-level.ttl
  • top-level.html

Em top-level.ttl, a tripla 

tm:sameAs owl:sameAs owl:sameAs .
é importante, por que ela liga minha base de conhecimento ao mundo dos dados ligados.

Relações em uma tabela

Abaixo temos informações na forma de tabela e na forma de sequência de palavras.

classepropriedade
grandezaunidade

A grandeza é uma classe. A unidade é uma propriedade. Uma classe tem uma propriedade. A unidade é uma propriedade da grandeza.

O cabeçalho da primeira coluna representa uma classe e o cabeçalho da segunda coluna representa uma propriedade.

Cada célula representa um objeto.

Na tabela abaixo, vou detalhar as relações.

grandezaunidade
comprimentometro
massaquilograma
temposegundo
correnteampere
temperaturakelvin

Em cada coluna, a relação entre o objeto e a classe é representada pela palavra é. Por exemplo, comprimento é grandeza e metro é unidade. Observe que o inverso da sequência, "grandeza é comprimento", é falso. A palavra é expressa uma relação vertical, de baixo para cima. Aqui segui o formato [o|a] <OBJETO> é [um|uma] <CLASSE>.

Quando vamos nos referir ao objeto, é opcional adicionar a classe antes. Por exemplo, o comprimento ou a grandeza comprimento.

Entre duas colunas, a relação entre objeto e propriedade é representada pelas palavras de e tem. Por exemplo, uma unidade do comprimento. Lembre que a palavra "do" e a contração das palavras "de o". Observe que também podemos formar sequências maiores, como: "uma unidade da grandeza comprimento", "o metro é uma unidade do comprimento" e "o metro é uma unidade da grandeza comprimento". A palavra de expressa uma relação diagonal de cima para baixo e para esquerda.

A palavra tem expressa uma relação horizontal da esquerda para direita. Por exemplo: grandeza tem unidade e comprimento tem unidade metro.

O desenho de uma tabela fica muito complicado ao aumentar as relações representadas. Então é melhor usar texto expressando as relações verticais e horizontais.

Base de estados

Esta base é uma lista ligada.

Exemplo: { "t": "", "v": "2", "%": "1" }
As chaves ("t" e "v") são os nomes das teclas ativas. Os valores ("" e "2") são nomes dos arquivos json que serão carregados ao pressionar as teclas ativas. O valor "" é especial, ele significa que a sequência acabou. A chave "%" é especial, seu valor contém o nome do arquivo anterior.

Exemplo: { "*": "ab123", "#": "3", "$": "3", "%": "1" }
As chaves "*", "#" e "$" são especiais. Seus valores contém respectivamente: o nome do arquivo da entidade na base de conhecimento, a quantidade de caracteres que o link envolve e o nome do próximo arquivo na base de estados.
